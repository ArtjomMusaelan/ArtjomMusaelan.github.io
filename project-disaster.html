Project 03

Natural Disaster Image Classifier

ML Engineer 98.3% Validation Accuracy CNN & XAI Deep Learning
Overview

AI-powered early warning system for disaster detection

Designed and developed an AI-powered early warning system to automatically classify natural disasters from images, supporting emergency response agencies in rapid disaster detection and resource mobilization.

Built end-to-end deep learning application collecting and processing ~300 curated images across floods and wildfires, implementing 4 CNN architecture iterations with systematic evaluation and explainable AI techniques.

The Challenge

Creating transparent and reliable disaster detection

Emergency response agencies needed automated disaster detection from visual data to enable faster response times and better resource allocation, but existing solutions lacked transparency and reliability.

The challenge required building a system that could accurately classify disasters while being interpretable and fair, addressing geographic underrepresentation and ensuring model decisions could be understood and trusted by emergency responders.

Technical Approach

Complete deep learning pipeline with explainable AI

Data Collection & Preprocessing

  • — Automated data collection using Selenium web scraper from DuckDuckGo, gathering ~300 curated images
  • — Implemented progressive data augmentation (rotation 20°-45°, shifting, shearing, zoom, brightness, flipping)
  • — Processed images at multiple resolutions (64×64 to 224×224) for architecture comparison
  • — Established multiple baselines including Random (50%), Human-level (98.3%), MLP (81.25%)

Model Development & Optimization

  • — Implemented 4 CNN iterations exploring complexity trade-offs and regularization techniques
  • — Best model: Simple 2-layer CNN achieving 98.3% validation accuracy and 87.5% test accuracy
  • — Explored dropout regularization, batch normalization, and VGG-style architectures
  • — Built custom MLP from scratch using NumPy for fundamental understanding

Explainable AI & Responsible Development

  • — LIME & KernelSHAP: Implemented multiple explainability techniques with parameter tuning (200-25000 samples)
  • — Segmentation Methods: Explored Quickshift, Felzenszwalb, SLIC, and Watershed techniques
  • — Fairness Analysis: Addressed geographic underrepresentation and image source diversity
  • — Human-Centered Design: Conducted think-aloud studies and A/B testing with wireframes in Figma
  • — Stakeholder Analysis: Comprehensive analysis with emergency agencies as primary stakeholder
  • — DAPS Framework: Transparent decision-making with documented bias mitigation
